# CS300

In this course, I worked on programs that focused on organizing and managing course data using different data structures. In Project One, I analyzed the run time and memory trade offs between vectors, hash tables, and binary search trees. In Project Two, I built a C++ advising program that loads course data from a file, stores it in a data structure, prints the courses in alphanumeric order, and allows users to search for a specific course and view its prerequisites.

The main problem I was solving was how to store and retrieve data efficiently. Instead of just writing code that worked, I had to think about performance. Understanding Big O notation and how each data structure handles insertion, searching, and traversal helped me see why structure matters. Choosing the right data structure directly affects how fast and scalable a program is.

One of the biggest challenges was debugging file input issues and making sure the data loaded correctly. I also had to work through sorting logic and make sure traversal functions behaved as expected. I overcame these roadblocks by testing smaller pieces of the program at a time and using output statements to trace what was happening. That process helped me become more confident in debugging instead of guessing.

This course changed how I approach designing software. I now think about how data will be used before I start coding. I also focus more on organizing my code into clear functions and writing it in a way that is readable and maintainable. Overall, this class strengthened my understanding of how algorithms and data structures impact programs and helped me write more intentional, structured code.
